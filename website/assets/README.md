# website/assets

This directory holds screenshot images for the Anchor subsite. It is populated
by the screenshot generators — do not hand-edit the images here.

## Dashboard screenshots (`dashboard-*.png`)

The teacher-dashboard shots (`dashboard-home.png`, `dashboard-session.png`,
`dashboard-bundles.png`, `dashboard-classes.png`, `dashboard-history.png`,
`dashboard-past-session.png`) are generated from the real Flutter Web dashboard
rendered against deterministic demo data — no backend, no real auth, no secrets.

Regenerate with one command:

```bash
cd ../../dashboard/screenshots
npm install            # first time only
node generate-screenshots.mjs
```

See [`dashboard/screenshots/README.md`](../../dashboard/screenshots/README.md)
for details.

## Agent screenshots (`agent-*.png`)

The student-facing agent shots (`agent-join-toast.png`,
`agent-block-overlay.png`, `agent-main-window.png`, `agent-tray-menu.png`) are
captured from the real agent's WinUI surfaces rendered against presentable demo
content — teacher "Ms Rivera", class code `PLINK-3B`, a readable allowlist,
matched to the dashboard's demo data so both sides tell one story. No backend,
no real auth, no secrets.

The agent's surfaces are pure WinUI 3 / DirectComposition, so they can't be
rendered to a file the way the Flutter dashboard can — they're driven through
the agent's `--show-test-*` self-tests and screenshotted via the integration
harness. Regenerate with one command (from the repo root, on Windows):

```powershell
./scripts/dev/generate-website-screenshots.ps1
```

That builds the agent, runs the opt-in `WebsiteScreenshots` generator (gated on
`ANCHOR_WEBSITE_SHOTS=1` so a routine test run never overwrites these), and
writes the four `agent-*.png` files here. See the "Visual enforcement" section
of [`agent/README.md`](../../agent/README.md) for the capture mechanics.

The browser-side shots are reused from `extension/store-listing/`
(`screenshot-block-1280x800.png`, `screenshot-popup-1280x800.png`) and are not
regenerated by either command.
